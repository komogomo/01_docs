⚠️ 注意事項・引き継ぎ事項

🔔 明日以降の開発で注意すべきこと

1. テナント分離の徹底

すべてのAPI実装で、必ず以下を確認してください：

  • JWTから取得したtenantIdを使用してデータをフィルタリング
  • 他のテナントのデータにアクセスできないことを確認
  • Prismaクエリで必ずwhere: { tenantId: user.tenantId }を含める

2. 認証・認可の適用

  • 公開すべきエンドポイント以外は、すべてJWTガードで保護される（デフォルト）
  • 管理者のみアクセス可能な操作には@Roles('admin')を追加（未実装）
  • @User()デコレーターでログインユーザー情報を取得

3. エラーハンドリング

  • 適切なHTTPステータスコードを返す（404, 401, 403, 400など）
  • ユーザーにわかりやすいエラーメッセージを返す
  • 機密情報をエラーメッセージに含めない

4. データバリデーション

  • DTOクラスで入力値を検証
  • class-validatorを使用したバリデーション
  • 日付・時間の範囲チェック

📝 環境変数の確認

以下の環境変数が.envファイルに設定されていることを確認してください：

# Database DATABASE_URL="postgresql://postgres:password@localhost:5432/securecity_db" # JWT JWT_SECRET="your-secret-key-here" JWT_EXPIRES_IN="1h" # Application NODE_ENV="development" PORT=3000 # Frontend URL (CORS) FRONTEND_URL="http://localhost:3001"
