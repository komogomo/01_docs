開発環境 再構築プロジェクト 作業記録

日付: 2025年10月16日

プロジェクト目標

既存の開発環境の混乱を解消し、Dドライブを主軸としたクリーンで再現性の高い開発環境をゼロから構築する。

フェーズ1: 環境クリーンアップとデータ移行

  • Cドライブの容量確保: Cドライブの空き容量不足（17.3GB）を特定し、環境再構築の方針を決定。
  • WSL (Ubuntu-24.04) の移行:
    • wsl --export コマンドでCドライブのUbuntuをバックアップ。
    • wsl --unregister コマンドでCドライブからUbuntuを完全に削除。
    • wsl --import コマンドでDドライブの指定フォルダ (D:\WSL\Ubuntu) に復元。
    • 結果: Cドライブの空き容量が 30.0GB まで回復。 完了
  • Docker Desktopのデータ保存先設定:
    • プログラム本体はCドライブにインストールし、容量を消費する仮想ディスク（Disk image）の保存先をDドライブに変更済みであることを確認。 完了

フェーズ2: 新規プロジェクトとデータベース(Docker)のセットアップ

  • 新規プロジェクト作成:
    • Dドライブに作業用フォルダ (D:\Projects) を作成。
    • npx @nestjs/cli new secure-city-api コマンドで、NestJSプロジェクトを新規作成。 完了
  • Docker環境の定義 (docker-compose.yml):
    • プロジェクトルートに docker-compose.yml を作成。
    • サービスとして PostgreSQL 16 (postgres:16-alpine) とDB管理ツール Adminer を定義。
    • データベースのデータ永続化先を D:/Projects/postgres-data に設定。 完了
  • 環境変数の設定 (.env):
    • プロジェクトルートに .env ファイルを作成。
    • DATABASE_URL をはじめ、JWTシークレットキーやポート番号などの設定情報を定義。 完了
  • データベースコンテナの起動:
    • docker-compose up -d コマンドを実行し、PostgreSQLとAdminerのコンテナを正常に起動。 完了

フェーズ3: Prismaによるデータベーススキーマの適用

  • Prismaのセットアップ:
    • npm install prisma @prisma/client でPrismaをプロジェクトにインストール。
    • npx prisma init で初期化し、prisma/schema.prisma ファイルを作成。 完了
  • スキーマ定義 (schema.prisma):
    • prisma/schema.prisma に、User, RefreshToken, Post, Commentモデルの定義を記述。
    • データソースとして、.env ファイルの DATABASE_URL を参照するように設定。 完了
  • データベースマイグレーション:
    • npx prisma migrate dev --name init コマンドを実行。
    • schema.prisma の定義に基づき、Docker上のPostgreSQLデータベース内に実際のテーブル群を自動で作成。 完了
