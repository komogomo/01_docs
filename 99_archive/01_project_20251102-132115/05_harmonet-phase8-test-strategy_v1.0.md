# HarmoNet Phase 8 — 試験方針書（Test Strategy）

## 1. 対象
- 実装済みすべての機能（掲示板・施設予約・アンケート・マイページ）

## 2. 試験区分
- UT（Unit Test）：実装機能の最小単位で動作確認
- IT（Integration Test）：画面内の1機能単位の結合確認
- PT（Procedure Test）：操作シナリオを通して総合確認

## 3. 担当
- AI：コード構造・ログ解析・動作トレース検証
- 人間：画面操作・翻訳・PDFプレビュー確認（TKD実施）

## 4. 合格基準
- 想定通りのログが出力されていること
- TKDによる操作試験で問題なく動作すること

## 5. 実施時期
- 実装完了後（Phase 9移行前）
- バグ修正はPhase 8内で完結
