## HarmoNet 掲示板モックアップ作成指示書（Claude 用）

### 1. コンテキスト

* プロジェクト名: HarmoNet
* 本番技術スタック: Next.js 16（App Router）+ React + Tailwind CSS
* 今回の目的: **UI 検証用の簡易モック** を作成し、画面構成・遷移・操作感を TKD と家族（WIFE）で確認すること。
* 前提:

  * 認証 / API / DB / RLS / i18n の本実装は一切不要。
  * すべてフロントエンドのみ、ダミーデータと `useState` で完結してよい。

このモックは **最終デザイン確定前のたたき台** です。本番コードとしての品質やディレクトリ構成は二の次で構いません。

---

### 2. ゴール（完成イメージ）

React で動作するシンプルな「掲示板モックアプリ」を作成してください。

* 画面は次の 3 つに限定します。

  1. **掲示板 TOP 一覧画面**（一覧 + 新規投稿ボタン）
  2. **掲示板詳細画面**（本文 + 翻訳 / 読上げアイコン + 返信）
  3. **新規投稿画面**（投稿区分トグル + タイトル + 本文 + PDF 添付ダミー）
* 最低限のルーティングと状態管理で、上記 3 画面を行き来できるようにしてください。
* 投稿はローカル state の配列に保持し、**擬似的に追加・表示される**程度で問題ありません。

---

### 3. 実装全体方針

* フレームワーク: 純粋な React で OK（Next.js を強く意識しなくてよい）。
* ルーティング:

  * 単一ファイル内で `useState<"top" | "detail" | "new">` を持ち、疑似遷移でもよい。
  * もしくは `react-router` などシンプルなルーターを使ってもよい（任意）。
* スタイル:

  * Tailwind 風のクラス名を付けてレイアウトしてください（実際のクラス内容は簡易で可）。
  * トーンは「白ベース・角丸・控えめ・シンプル」程度を意識。
* データ:

  * 初期投稿データ、返信データはコード内にハードコードしてよい。
  * 投稿追加時はローカル state を更新するだけでよい。
* 非機能:

  * サーバ通信・DB 接続・認証処理は一切行わない。
  * i18n ライブラリは使わず、日本語固定テキストで構わない。

---

### 4. 画面別仕様

#### 4-1. 掲示板 TOP 一覧画面

**役割:** 投稿一覧の表示 + 新規投稿画面への入口。

1. 一覧カード表示

   * ダミー投稿を複数件表示してください。
   * 1 件あたりのカード構成:

     * 1 行目: 「投稿区分タグ」1 つ

       * 例: `[重要]`, `[お知らせ]`, `[北-A]` など
     * 2 行目: タイトル
     * 3 行目:

       * 左側: 投稿者種別（例: `管理組合`, `一般利用者`）
       * 右側: 投稿日時（例: `2025/11/21 10:30`）
     * 添付ファイルがある場合のみ、カード右上あたりに **クリップアイコン** を表示。

2. ページ下部のコントロール行（1 行構成）

   * 画面下に 1 行のバーとして配置。
   * 左端: `[10件 ▼]`（表示件数ドロップダウン）

     * 文言ラベル「表示件数」は付けない。
     * 選択肢は `10件 / 20件 / 50件` 程度（モックでは見た目だけで可）。
   * 右側: ページネーション

     * `[前へ] [1] [2] [3] [次へ]`
   * モックでは、クリック時は `console.log` するだけでよい（ページ切替ロジックは不要）。

3. 新規投稿ボタン（フローティング FAB）

   * 画面右下にフローティングの丸い「＋」ボタンを表示。
   * スクロールしても常に右下に固定。
   * クリックすると **新規投稿画面** に遷移。
   * ボタンテキストは不要で、アイコンのみでよい（ホバー時に `title="新規投稿"` を付与）。

4. 遷移

   * カードクリック → 対応する「掲示板詳細画面」に遷移。
   * 右下の「＋」ボタン → 「新規投稿画面」に遷移。

---

#### 4-2. 掲示板詳細画面

**役割:** 1 件の投稿内容 + 添付ファイル + 返信一覧・返信入力を表示。

1. 投稿内容ブロック

   * 上から順に以下を表示:

     1. 投稿区分タグ + タイトル
     2. 投稿者種別 + 投稿日時 + 対象範囲

        * 例: `管理組合 / 北-Aグループ向け  2025/11/21 10:30`
     3. 投稿本文（複数行テキスト）

2. 本文直下のアクションアイコン（翻訳 / 読上げ）

   * 本文のすぐ下に、右寄せでアイコン 2 つを横並びで配置。
   * 使用アイコン（Lucide 想定）:

     * 翻訳: `Languages`
     * 読上げ: `Volume2`
   * アイコンのみ表示（テキストラベルなし）。
   * アクセシビリティ:

     * 翻訳アイコン:

       * `aria-label="投稿本文を翻訳"`
       * `title="翻訳"`
     * 読上げアイコン:

       * `aria-label="投稿本文を読み上げ"`
       * `title="読み上げ"`
   * 動作（モック仕様）:

     * クリックで ON/OFF トグル。
     * ON の間は色を変えるなどして状態が分かるようにする。
     * 実際の翻訳 / TTS は不要で、`console.log` するだけでよい。

3. 添付ファイル一覧

   * 「添付ファイル」見出しの下に、PDF ファイル名リストを表示。
   * 各行に PDF アイコン + ファイル名程度。
   * クリック時の動作（モック）:

     * `alert("PDFプレビュー（モック）")` 程度でよい。

4. 返信セクション

   * 見出し: 「返信」
   * 返信一覧:

     * 新しい順で複数件表示。
     * 各返信の表示:

       * 1 行目: 表示名 + 右側に日時

         * 表示名は `匿名` または `ニックネーム`（実名は出さない）。
       * 2 行目以降: 返信本文テキスト
   * 返信入力エリア:

     * テキストエリア（プレースホルダ: `返信を入力…`）
     * 表示名の規定表示チェックボックス（見た目だけで可）

       * 例: `□ 常に匿名で返信する`
     * 「送信」ボタン

       * クリック時:

         * テキストエリア内容を返信として一覧の先頭に追加（表示名は `匿名` 固定で構わない）。
         * 送信後はテキストエリアをクリア。

5. 戻る導線

   * 画面上部に「＜掲示板TOPへ」リンクを配置し、クリックで TOP 一覧画面へ戻る。

---

#### 4-3. 新規投稿画面（/bbs/new 想定）

**役割:** 新しい掲示板投稿を作成するためのフォーム画面。

1. 入力項目

   * 投稿区分 *

     * 単一選択のトグルボタン群として表示。
     * 例（管理組合ユーザ想定）:

       * `[重要] [お知らせ] [ルール] [北-A] [北-B]`
     * 一度に 1 つだけ選択可能。
     * 初期状態では何も選択されていない状態でもよい。
   * タイトル *

     * 1 行テキスト入力。
   * 本文 *

     * リッチテキスト“風”の入力エリア。
     * 実際のリッチエディタは不要ですが、上部に簡易ツールバーを表示して「それっぽく」してください。

       * 例: 太字アイコン, 箇条書きアイコン（押したら `console.log` するだけで可）。
     * 入力自体は通常の `<textarea>` または `contentEditable` div で十分です。
   * 添付ファイル（任意）

     * 「PDF ファイルを選択」程度の UI（`<input type="file">`）。
     * 実際のアップロードやプレビュー処理は不要です。
   * ボタン

     * 「投稿」ボタン
     * 「キャンセル」ボタン（掲示板TOPに戻る）

2. 必須表示とバリデーション

   * ラベル右に赤 `*` を付与:

     * `投稿区分 *`
     * `タイトル *`
     * `本文 *`
   * 画面下部に小さく「* は必須項目です。」と表示。
   * バリデーション（モック仕様）:

     * 投稿区分が未選択、またはタイトル / 本文が空のときは「投稿」ボタンを `disabled` にする動作が望ましいです。
     * もしくは、ボタンは押せるが、押したときに `alert("必須項目を入力してください")` を出す程度でも構いません。

3. 投稿前確認ポップアップ

   * 必須チェックを通過した場合、「投稿」ボタン押下時に `window.confirm` を使って確認モーダルを表示してください。
   * 文言イメージ:

     ```
     掲示板に以下の内容で投稿します。よろしいですか？

     投稿区分: [重要]
     タイトル: ○○○○
     本文: △△△△（先頭数十文字のみでOK）
     ```
   * `OK` → 投稿処理続行。
   * `キャンセル` → 何もしない。

4. 投稿完了後の挙動

   * `OK` で投稿確定時:

     * ローカル state の投稿配列に新規投稿を先頭追加。
     * 掲示板TOP 一覧画面に遷移。
     * TOP では新規投稿が一番上に見えること。

5. キャンセル

   * 「キャンセル」ボタン押下時:

     * 掲示板TOP 一覧画面に遷移。
     * 入力内容は破棄してよい（ドラフト保持は不要）。

---

### 5. データモデル（モック用の簡易形）

厳密な型定義は不要ですが、イメージが共有しやすいように簡易モデルを示します。

```ts
type PostCategory = "重要" | "お知らせ" | "ルール" | "北-A" | "北-B" | "フリー" | string;

type AuthorType = "管理組合" | "一般利用者" | "テナント管理者" | string;

interface AttachmentMock {
  id: string;
  name: string; // PDF ファイル名
}

interface ReplyMock {
  id: string;
  displayName: string; // "匿名" or ニックネーム
  body: string;
  createdAt: string; // 日時文字列
}

interface PostMock {
  id: string;
  category: PostCategory;      // 投稿区分タグ
  authorType: AuthorType;      // 投稿者種別
  targetLabel: string;         // 対象範囲ラベル（例: "北-Aグループ向け"）
  title: string;
  body: string;
  createdAt: string;
  hasAttachment: boolean;
  attachments?: AttachmentMock[];
  replies: ReplyMock[];
}
```

この程度の構造があれば十分です。実装時に多少変形しても問題ありません。

---

### 6. 実装上のお願い

* コードは 1 プロジェクト or 1 ファイルにまとまっていて構いません。
* `useState` と簡単なコンポーネント分割だけで実装してください。

  * 例: `BoardTop`, `BoardDetail`, `NewPostForm` など。
* 重要なのは「画面構成と遷移のイメージ」です。

  * 見た目は HarmoNet のトーンに寄せつつも、厳密なデザインは不要です。
* コメントで主要な箇所に補足を書いてもらえると、読み解きやすくなります。
