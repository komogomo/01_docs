## 1. 概要

本ドキュメントは、**HarmoNet アプリケーションの共通フッター領域**の基本仕様を定義する。

### 1.1 適用範囲

* 対象画面

  * ログイン画面を除く、HarmoNet の主要画面全般
* 表示位置

  * 画面最下部（モバイルでは固定表示を基本とし、レイアウト詳細は `ch04_AppFooter_v1.0` に従う）

### 1.2 設計原則

* **アクセス性**: 主要機能へのアクセスを少ないタップ数で実現する
* **視認性**: 現在の画面を明確に示し、迷子になりにくいナビゲーションとする
* **一貫性**: すべての画面で同じレイアウト・デザインルールを適用する
* **静けさ**: 「やさしい・自然・控えめ」な UI トーンを保ち、過度な装飾やアニメーションは避ける

---

## 2. 基本仕様

### 2.1 レイアウト

フッターは、主要機能へのショートカットを水平に並べたナビゲーションバーとする。

表示例（概念図）:

```
┌───────────────────────────────────────┐
│  [ホーム]  [お知らせ]  [掲示板]  [駐車場]  [マイページ]  [ログアウト] │
└───────────────────────────────────────┘
```

実際の構造・クラス構成は、`ch04_AppFooter_v1.0` および `ch05_FooterShortcutBar_v1.0` の詳細設計に従う。

### 2.2 寸法（基準値）

| 項目      | 値                          | 備考                                |
| ------- | -------------------------- | --------------------------------- |
| 高さ      | 約 60px                     | モバイルを基準とした目安値                     |
| 幅       | 100%                       | 画面幅いっぱい                           |
| 内部パディング | 上下 8px 程度                  | 実装では Tailwind の `py-2`〜`py-3` を想定 |
| 背景色     | `bg-white`                 | 共通デザインシステムに準拠                     |
| 上部ボーダー  | `border-t border-gray-200` | 共通デザインシステムに準拠                     |

正確な Tailwind クラスおよびレスポンシブ挙動は詳細設計書側で定義する。

## 3. ナビゲーションボタン

### 3.1 ボタン一覧（MVP 想定）

| No | 機能名   | 概要                 | 遷移先の例      |
| -- | ----- | ------------------ | ---------- |
| 1  | ホーム   | トップダッシュボード画面       | `/home`    |
| 2  | お知らせ  | お知らせ一覧画面           | `/notices` |
| 3  | 掲示板   | 掲示板一覧画面            | `/boards`  |
| 4  | 駐車場   | 駐車場予約関連画面          | `/parking` |
| 5  | マイページ | 利用者の個人設定・プロフィール    | `/mypage`  |
| 6  | ログアウト | セッション終了後にログイン画面へ遷移 | `/login`   |

実際のルーティングは Next.js のルート設計に従う。

### 3.2 ボタンの基本仕様（デザイン観点）

* 各ボタンは、アイコン＋ラベルの縦配置を基本とする。
* 横方向は等間隔に並べ、画面幅に応じて自動調整する。
* タップ領域は十分な大きさ（目安: 44x44px 以上）を確保する。
* アイコンは線形アイコン（outline）を使用し、

  * 非アクティブ: サブテキストカラー（例: `text-gray-500`）
  * アクティブ: アクセントカラー（例: `text-primary-500`）
    として視覚的階層をつける。

### 3.3 多言語対応

各ボタンのラベルは i18n キーで管理し、選択中の言語に応じて切り替える。

* ラベルの文言は、HarmoNet の i18n 設計（StaticI18nProvider 等）に従う。
* 本書ではキー命名規則を固定せず、詳細は共通 i18n 設計書に委ねる。

## 4. アクティブ状態

### 4.1 アクティブ状態の定義

* 現在表示中の画面に対応するボタンを「アクティブ状態」として強調する。
* ログアウトボタンはアクティブ状態を持たない（常に通常表示）。

### 4.2 アクティブ時のスタイル（方針）

| 項目    | 通常時             | アクティブ時（例）                     |
| ----- | --------------- | ----------------------------- |
| 背景色   | 透明              | 薄いアクセント背景（例: `bg-primary-50`） |
| アイコン色 | `text-gray-500` | `text-primary-500`            |
| ラベル色  | `text-gray-500` | `text-primary-600`            |
| ラベル太さ | `font-normal`   | `font-semibold`               |

具体的なクラス指定は `ch04_AppFooter_v1.0` / `ch05_FooterShortcutBar_v1.0` にて定義する。

---

## 5. ログアウトボタン

### 5.1 位置と扱い

* フッター右端に配置する。
* アクティブ状態は持たず、常に通常スタイルとする。

### 5.2 挙動（概要）

1. ログアウトボタンが押下されたら、確認ダイアログ（ブラウザ標準／モーダル等）でユーザーに確認する。
2. ユーザーが確定した場合、セッション情報を破棄し、ログイン画面へ遷移する。
3. 挙動の詳細は認証・セッション管理の設計書に従う。

---

## 6. アクセシビリティ

* フッター全体には適切な landmark ロール（例: `role="contentinfo"`）を付与する。
* ナビゲーションボタンはキーボードで順番にフォーカスできるようにし、Enter / Space で実行可能とする。
* 現在ページに対応するボタンには `aria-current="page"` 等を付与し、スクリーンリーダーに伝える。
* アイコンには `aria-hidden="true"` を付与し、ラベルテキストで意味が伝わるようにする。

アクセシビリティの詳細要件は、共通アクセシビリティ指針および各コンポーネントの詳細設計書にて補完する。

---

## 7. インタラクション（概要）

* Hover / Focus 時

  * 背景色をわずかに変化させ、フォーカスリングを明示する。
  * 過度なアニメーションは用いず、150ms 程度のフェードを上限とする。
* タップ時

  * モバイルでは一時的に背景色を変えるなど、軽いフィードバックを与える。

詳細なインタラクション仕様は、AppFooter / FooterShortcutBar の詳細設計に委ねる。
