## 1. 目的

本書は、HarmoNet アプリケーション全体に共通する **UI デザインの基準（Design System）** を定義する基本設計書である。

* すべての画面・コンポーネントが従うべき **色・タイポグラフィ・余白・角丸・影** のルールを定義する。
* AppHeader / AppFooter / 共通コンポーネント / 各画面詳細設計の「上位レイヤ」として参照される。
* 実装時の Tailwind クラス指定・トークン定義は、本書の方針に従って各詳細設計書で具体化する。

> 本書は HarmoNet 専用の設計書であり、旧「セキュレアシティ」系プロジェクトの名称・ID・前提は一切含まない。

## 2. デザインコンセプト

### 2.1 トーン＆マナー

* やさしい・自然・控えめ
* Apple カタログ風のミニマルなプロダクト感
* 情報量を詰め込みすぎず、余白で「呼吸」できる画面構成

### 2.2 UI の特徴

* 白ベースの背景に、わずかな影と柔らかい角丸（`rounded-2xl`）
* 線形アイコン（outline）と、必要最小限のアクセントカラー
* ボタンはフラット寄りだが、状態変化（hover / focus / disabled）でわかりやすさを確保

## 3. タイポグラフィ

### 3.1 基本フォント方針

* 「可読性の高いゴシック系サンセリフ」を標準とし、**特定フォントを必須とはしない**。
* 実装上は各 OS のシステムフォントスタック（`system-ui` など）を使用する。
* Windows 環境で BIZ UDゴシック が利用可能な場合は、推奨フォント候補として優先指定してよい（必須ではない）。

### 3.2 フォントサイズと階層

| 用途       | Tailwind 目安            | 説明           |
| -------- | ---------------------- | ------------ |
| 画面タイトル   | `text-2xl md:text-3xl` | 画面最上部の主タイトル  |
| セクション見出し | `text-xl`              | カードやブロックの見出し |
| 本文       | `text-base`            | 一般的な説明文      |
| 補足・注意書き  | `text-sm`              | 補助的なテキストや注釈  |

* 太さは `font-semibold` を見出しの基本とし、本文は `font-normal`。
* 行間は `leading-relaxed` を標準とする。

## 4. カラーパレット

### 4.1 基本カラー

| 種別        | Tailwind 例        | Hex 目安  | 用途            |
| --------- | ----------------- | ------- | ------------- |
| 背景（アプリ全体） | `bg-gray-50`      | #FAFAFA | 画面全体のベース背景    |
| カード背景     | `bg-white`        | #FFFFFF | コンテンツカード・モーダル |
| 枠線        | `border-gray-200` | #E5E7EB | カード・入力欄の境界線   |
| 本文        | `text-gray-800`   | #1F2937 | 通常テキスト        |
| サブテキスト    | `text-gray-500`   | #6B7280 | 補足・説明テキスト     |

### 4.2 アクセントカラー（仮）

* プライマリカラー（ボタン・リンク）

  * Tailwind 想定: `primary-500` 系（実際の色コードはデザイン確定後に設定）
* ステータスカラー

  * 成功: `green-500`
  * 警告: `yellow-500`
  * エラー: `red-500`

> ステータスカラーは主にテキストやアイコン、細いボーダーで使用し、背景塗りつぶしは抑制する。

## 5. 余白・角丸・影

### 5.1 余白（Spacing）

* コンテンツ上下余白: `py-6`〜`py-10`
* カード内パディング: `p-4 md:p-6`
* フォーム要素の縦間隔: `space-y-4` を基本とする。
* ボタンとフォーム本体の距離: `mt-4`。

### 5.2 角丸（Radius）

* カード・モーダル: `rounded-2xl`
* 入力欄・ボタン: `rounded-xl`
* アイコンボタン等: `rounded-full` も許容

### 5.3 影（Shadow）

* デフォルト: `shadow-sm`
* Hover 時の上限: `shadow-md`
* 大きな影（`shadow-lg` 以上）は原則使用しない。

## 6. コンポーネント共通ルール

### 6.1 カード

* 背景: `bg-white`
* 角丸: `rounded-2xl`
* 影: `shadow-sm`
* 内側余白: `p-4 md:p-6`
* 上下余白: 親レイアウトで `space-y-6` を基本とする。

### 6.2 ボタン

* Primary ボタン

  * 背景: `bg-primary-500`
  * テキスト: `text-white`
  * 角丸: `rounded-xl`
  * フォーカス: `focus:outline-none focus:ring-2 focus:ring-primary-500`
* Secondary ボタン

  * 背景: `bg-white`
  * 枠線: `border border-gray-300`
  * テキスト: `text-gray-800`

### 6.3 入力欄

* 背景: `bg-gray-50`
* 枠線: `border border-gray-300`
* 角丸: `rounded-xl`
* フォーカス時

  * 枠線色をプライマリに変更し、`ring` を付与
* プレースホルダテキスト: `text-gray-400`

---

## 7. アイコン・イラスト

* アイコンは線形（outline）アイコンセットを利用（例: `lucide-react`）。
* サイズは `20px`〜`24px` を基本とし、テキストと過度に競合しないようにする。
* 装飾的なイラストは最小限とし、読みやすさを優先する。

---

## 8. レスポンシブ方針

* すべての画面はモバイルファーストで設計する。
* スマートフォン: 1 カラム構成を基本とし、横スクロールは避ける。
* タブレット以上: 必要に応じて 2 カラム以上のレイアウトを許容するが、

  * コンテンツ密度を上げすぎない
  * 余白と読みやすさを優先

共通レイアウトの詳細は `common-contents-layout_v1.2.md` を参照する。

---

## 9. アクセシビリティとの関係

* 色のコントラスト比は、主要なテキスト・ボタンについて WCAG 2.1 AA 準拠を目標とする。
* フォーカスリングは必ず可視化し、CSS で消さない。
* エラー表示（ErrorAlert / InlineFieldError）は、色だけでなくアイコンやテキストで状態を伝える。

アクセシビリティの詳細要件は、各コンポーネント／画面の詳細設計および今後作成する共通アクセシビリティ指針にて定義する。
