## 4.1 機能タイルの役割

HOME 画面の機能タイルは、ログイン後に利用者が頻繁に利用する機能への
「入り口」を 2×3 のカードタイルとしてまとめたものである。

* HOME を「機能ポータル」として機能させることが目的。
* 掲示板機能は 1つの実体（掲示板TOP）に対し、
  タイルごとに異なるフィルタ状態（ALL / NOTICE / RULES）で遷移する想定とする。
* MVP 段階では、見た目を優先し、リンクを無効（onClick なし）とすることを許容する。

雑談板は設けず、掲示板は「公式情報のハブ」として扱うため、
機能タイルも公式機能のショートカットに限定する。

## 4.2 タイル構成と配置

### 4.2.1 配置方針

機能タイルは、AppFooter のナビゲーションと意味・優先度を揃えるため、
次の順序で 2×3 のグリッドに配置する。

* 上段：

  1. お知らせ
  2. 掲示板
  3. 施設予約
* 下段：
  4. 運用ルール
  5. 通知設定
  6. ダミー（将来機能枠）

HOME では、フッターと同じ順番で「お知らせ」「掲示板」「施設予約」を上段にまとめることで、
利用者が位置関係を自然に理解しやすい構成とする。

### 4.2.2 タイル一覧（概要）

| ID | ラベル   | 役割概要                       |
| -- | ----- | -------------------------- |
| T1 | お知らせ  | 管理組合からのお知らせ一覧（NOTICE フィルタ） |
| T2 | 掲示板   | 全掲示情報の一覧（ALL フィルタ）         |
| T3 | 施設予約  | 共用施設の予約一覧・予約申請             |
| T4 | 運用ルール | 管理規約・運用ルールの一覧              |
| T5 | 通知設定  | PUSH / メール通知の受信設定          |
| T6 | ダミー   | 将来機能用のプレースホルダ              |

各タイルの詳細仕様（表示テキスト・アイコン・遷移先）は、以下の節で定義する。

## 4.3 タイル共通仕様

### 4.3.1 ワイヤーフレーム

各タイルは、共通のカードスタイルで表現する。

```
┌────────────────┐
│   [アイコン]                │
│   ラベルテキスト            │
│   説明テキスト（1〜2行）   │
└────────────────┘
```

* アイコン：lucide-react の線形アイコンを使用する。
* ラベル：機能名（例：「お知らせ」「掲示板」など）。
* 説明：その機能で何ができるかを 1〜2 行で簡潔に表現する。

### 4.3.2 ビジュアル・インタラクション

* カードは角丸 2xl、控えめなシャドウ、余白多めのミニマルなスタイルとする。
* ラベル・説明テキストは、BIZ UD ゴシック系フォントを前提とする。
* MVP 段階では、`isEnabled = false` としてリンク無効状態で実装してよい。

  * `cursor: default`、ホバー時の色変化なし。
  * 将来的に機能実装が完了したタイミングで `isEnabled = true` とし、
    `<Link>` や `onClick` を有効化する。

## 4.4 各タイルの仕様

### 4.4.1 T1：お知らせタイル

* ラベル：

  * 「お知らせ」（Static i18n キーで管理）
* 想定アイコン：

  * 例）`Bell` または `Megaphone`（lucide-react）
* 役割：

  * 掲示板TOP を NOTICE フィルタ状態で表示するためのショートカット。
  * HOME 上部のお知らせセクションと連携し、
    「お知らせの続き・一覧を確認する」入口として機能させる。
* 遷移先（将来）：

  * 例）`/board?tag=NOTICE`
* 備考：

  * AppFooter に「お知らせ」タブを設ける場合は、同じ遷移先に揃える。

### 4.4.2 T2：掲示板タイル

* ラベル：

  * 「掲示板」（Static i18n キー）
* 想定アイコン：

  * 例）`MessageSquare`（lucide-react）
* 役割：

  * 掲示板TOP を ALL フィルタ状態で表示するショートカット。
  * テナント内のすべての掲示情報（お知らせ・運用連絡など）を一覧表示する。
* 遷移先（将来）：

  * 例）`/board?tag=ALL` または `/board`（デフォルトが ALL の場合）
* 備考：

  * AppFooter の「掲示板」アイコンも同じ遷移先とし、役割を統一する。

### 4.4.3 T3：施設予約タイル

* ラベル：

  * 「施設予約」（Static i18n キー）
* 想定アイコン：

  * 例）`Calendar` または `CalendarClock`
* 役割：

  * 共用施設（集会所、ゲストルームなど）の予約状況確認・予約申請画面への入口。
* 遷移先（将来）：

  * 例）`/facility` または `/facility/reservations`
* 備考：

  * 施設予約機能の詳細仕様は、施設予約の別設計書で定義する。

### 4.4.4 T4：運用ルールタイル

* ラベル：

  * 「運用ルール」（Static i18n キー）
* 想定アイコン：

  * 例）`FileText` または `BookOpen`（規約・ルールのイメージ）
* 役割：

  * 管理規約や運用ルールを確認するための入口。
  * 掲示板の RULES タグ付き投稿を一覧表示するか、
    ルール専用画面へ遷移させる。
* 遷移先（候補）：

  * 例1）`/board?tag=RULES`（掲示板フィルタで実現）
  * 例2）`/rules`（ルール専用画面）
* 方針：

  * 現時点では実装方式は未確定とし、「RULES カテゴリのルールを参照する入口」であることのみを定義する。
  * 実装方式が決まり次第、本タイルの遷移先 URL を確定させる。

### 4.4.5 T5：通知設定タイル

* ラベル：

  * 「通知設定」（Static i18n キー）
* 想定アイコン：

  * 例）`BellDot` または `Settings`
* 役割：

  * PUSH 通知やメール通知の ON/OFF、通知対象カテゴリなどを設定する画面への入口。
* 遷移先（将来）：

  * 例）`/settings/notifications`
* 備考：

  * 実際の通知チャネル・カテゴリの詳細仕様は、通知機能の設計書側で定義する。

### 4.4.6 T6：ダミータイル

* ラベル：

  * 「準備中」など（Static i18n キー）
* 想定アイコン：

  * 例）`MoreHorizontal` または `PanelsTopLeft`
* 役割：

  * 将来追加予定の機能を配置する枠として扱うプレースホルダ。
* 遷移先：

  * なし（リンク無効）
* 挙動：

  * タップしても何も起きない。
  * 将来、具体的な機能が決まり次第、ラベル・説明・遷移先を更新する。

## 4.5 アイコン対応表（案）

lucide-react を前提としたアイコン対応表の案を示す。
最終決定は UI 実装段階で微調整してよいが、意味が大きく変わらないことを前提とする。

| タイルID | ラベル   | lucide-react アイコン候補               |
| ----- | ----- | --------------------------------- |
| T1    | お知らせ  | `Bell`, `Megaphone`               |
| T2    | 掲示板   | `MessageSquare`                   |
| T3    | 施設予約  | `Calendar`, `CalendarClock`       |
| T4    | 運用ルール | `FileText`, `BookOpen`            |
| T5    | 通知設定  | `BellDot`, `Settings`             |
| T6    | ダミー   | `MoreHorizontal`, `PanelsTopLeft` |

実装時は、HarmoNet 全体のアイコン使用ルールと整合を取りつつ、
同種のアイコンを極力再利用する方針とする。

## 4.6 AppFooter との整合

* AppFooter の「掲示板」アイコンは、掲示板TOP（ALL フィルタ）へのショートカットとする。
* 将来的に AppFooter に「お知らせ」「施設予約」などを追加する場合は、
  HOME の機能タイルと遷移先を揃える。
* HOME の機能タイルは、AppFooter を補完する「視覚的な機能ポータル」として位置づける。

---

## 4.7 本章まとめ

* HOME 画面の機能タイルは、2×3 グリッドで

  * 上段：お知らせ／掲示板／施設予約
  * 下段：運用ルール／通知設定／ダミー
    を配置する。
* 掲示板関連タイル（お知らせ・掲示板・運用ルール）は、
  いずれも掲示板TOPのフィルタ状態（NOTICE / ALL / RULES）を変えて表示するショートカットとして設計する。
* MVP 段階では、すべてのタイルのリンクを無効とし、
  見た目の統一とレイアウトの確認を優先する。
* 遷移先 URL や実際のフィルタキーは、掲示板・施設予約・通知設定など各機能の詳細設計にて確定させる。
