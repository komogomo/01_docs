## 2.1 コンテンツ領域の構成（2セクション構成）

HOME 画面のコンテンツ領域は、以下 **2つのセクションのみ** で構成する。
ウェルカムセクションは廃止し、情報提示を最優先にした構造へ再編している。

```
┌───────────────────────────┐
│  【AppHeader】                      │ ← 共通部品（対象外）
├───────────────────────────┤
│  ① お知らせセクション（最新3件）           │
├───────────────────────────┤
│  ② 機能タイルセクション（6タイル）         │
├───────────────────────────┤
│  【FooterShortcutBar / AppFooter】  │ ← 共通部品（対象外）
└───────────────────────────┘
```

本基本設計書が扱うのは **①お知らせ** と **②機能タイル** のみであり、
ヘッダー／フッター等は共通部品の各設計書に従う。

## 2.3 セクション概要

### ① お知らせセクション（最新3件）

* 役割：重要なお知らせの即時提示
* 表示内容：最新3件、カテゴリバッジ、未読バッジ、投稿日
* アクション：カードタップ→お知らせ詳細へ
* データスコープ：`tenant_id` によるフィルタ

### ② 機能タイルセクション（6タイル / MVP版）

* 役割：主要機能へのエントリーメニュー
* タイル構成（上段→下段）：

  1. 施設予約
  2. 掲示板
  3. 回覧板
  4. お知らせ
  5. 通知設定
  6. ダミー（準備中）
* アイコン：Lucide Icons（名称のみ基本設計で定義）
* レイアウト：

  * スマートフォン：2×3
  * タブレット/PC：3×2

---

## 2.4 セクション間の関係性

```
お知らせセクション
    ↓（最新情報を視認）
機能タイルセクション
    ↓（行動へ誘導）
```

* HOME は **情報 → 行動** の2段階モデル
* 各セクションは独立し、片方の障害がもう一方へ影響しない
* UI トーン・フォント・マージンは HarmoNet Design System に従って統一

---

## 2.5 レスポンシブ方針（簡易仕様 / 基本設計レベル）

| デバイス    | ブレークポイント   | 主なレイアウト         | 備考         |
| ------- | ---------- | --------------- | ---------- |
| PC      | ≥1024px    | 最大幅 1024px、中央寄せ | コンテンツ幅制御のみ |
| タブレット   | 768–1023px | 機能タイル 3×2       | UIトーン維持    |
| スマートフォン | <768px     | 機能タイル 2×3       | タップ領域を広く確保 |

**共通調整:**

* セクション間の余白は HarmoNet 基準（16px/24px）
* フォントは BIZ UD ゴシック相当を使用（詳細設計で扱う）
