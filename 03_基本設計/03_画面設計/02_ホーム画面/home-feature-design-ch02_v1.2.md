## 2.1 コンテンツ領域のセクション構成

HOME 画面のコンテンツ領域は、以下 **2つのセクションのみ** で構成する。
ウェルカムメッセージ等の装飾的要素は設けず、「重要情報の提示」と「機能ポータル」を最優先とする。

1. セクションA：最新のお知らせ
2. セクションB：機能タイル（2×3 グリッド）

AppHeader / AppFooter は共通部品として別紙で定義し、本章では扱わない。

---

## 2.2 レイアウトイメージ

スマートフォン縦画面（想定幅 360〜430px）を前提に、概ね 1画面内に
「最新のお知らせ」＋「機能タイル 2×3」が収まるレイアウトとする。

```
┌───────────────────────────┐
│ AppHeader（共通部品）                           │
├───────────────────────────┤
│ セクションA：最新のお知らせ                     │
│  ┌ お知らせカード[1]                         ┐ │
│  └ お知らせカード[2]（最大3件まで可変）      ┘ │
│                                               │
│ （0件時は「現在表示するお知らせはありません。」） │
├───────────────────────────┤
│ セクションB：機能タイル 2×3                    │
│  [お知らせ] [掲示板] [施設予約]                │
│  [運用ルール] [通知設定] [ダミー]             │
├───────────────────────────┤
│ AppFooter（共通部品）                           │
└───────────────────────────┘
```

* スクロール方向は縦のみ。
* タブ UI やサイドメニューは設けず、HOME は 1カラム構成とする。

## 2.3 セクションA：最新のお知らせ

### 2.3.1 目的

* 管理組合からの重要なお知らせ（NOTICE タグ付き掲示）を、
  HOME 上部で簡潔に提示する。
* 利用者が HOME を開いたタイミングで、
  重要なお知らせの有無が一目で分かることを目的とする。

### 2.3.2 表示件数と高さ

* デフォルト表示件数：**2件**
* テナント設定により 1〜3件の範囲で変更可能とする。

  * UI 崩れ防止のため、HOME 上での表示件数は最大 3 件とする。
* お知らせカード 1件あたりの高さは、タイトル 2行程度で折り返し、
  セクション全体が画面の約 1/3〜1/2 に収まることを目安とする。

### 2.3.3 カード構造（ワイヤーフレーム）

```
┌─────────────────────────┐
│ [バッジ: お知らせ]   2025/11/21              │
│ タイトルタイトルタイトルタイトル…（最大2行）│
│                                       ＞   │
└─────────────────────────┘
```

* 上段：

  * 左に小さなバッジ（例：「お知らせ」）
  * 右に日付（`YYYY/MM/DD`）
* 中段：

  * タイトルテキスト（最大 2 行、超過分は「…」で省略）
* 右端：

  * 「＞」アイコン、または「詳細」テキストボタン

### 2.3.4 0件時の表示

* NOTICE タグ付きの公開中投稿が 1件もない場合：

  * セクションタイトルの直下に

    * 「現在表示するお知らせはありません。」
      のテキストのみを表示し、カードは描画しない。

## 2.4 セクションB：機能タイル

### 2.4.1 目的

* 利用頻度の高い機能へのショートカットを、
  2×3 のカードタイルとして一覧表示する。
* 掲示板機能は 1つの実体（掲示板TOP）に対し、
  タイルごとに異なるフィルタ状態（ALL / NOTICE / RULES）で遷移する想定とする。

### 2.4.2 タイル配置

HOME の機能タイルは、AppFooter のナビゲーションとの整合を重視し、
以下の順序で配置する。

* 上段：

  1. お知らせ
  2. 掲示板
  3. 施設予約
* 下段：
  4. 運用ルール
  5. 通知設定
  6. ダミー（将来機能枠）

各タイルの細かな説明文・アイコン・遷移先 URL は ch04 で定義し、
本章では「配置」と「役割の違い」のみを扱う。

### 2.4.3 タイルのワイヤーフレーム

各タイルは、共通のカードスタイルで表現する。

```
┌───────────────┐
│  [アイコン]              │
│  ラベルテキスト          │
│  説明テキスト（1〜2行） │
└───────────────┘
```

* 1行目：アイコン（lucide-react）
* 2行目：ラベル（例：「お知らせ」「掲示板」）
* 3行目：簡単な説明（例：「重要なお知らせの一覧」など）

MVP 段階では、見た目を最優先し、タイル全てのリンクを無効とすることを許容する。
将来、実装完了に合わせて `<Link>`／`onClick` を有効化する。

## 2.5 掲示板との関係（ビューとしての整理）

### 2.5.1 掲示板の役割

* 掲示板TOP は、テナント内の全ての掲示情報（お知らせ・運用連絡など）を
  一覧表示する画面とする。
* 雑談用途の「雑談板」は設けず、掲示板はすべて「公式な情報」を扱う場とする。

### 2.5.2 HOME との連携イメージ

* セクションA（最新のお知らせ）

  * 掲示板の NOTICE タグ付き投稿の一部をチラ見せするビュー。
* セクションB（機能タイル）

  * お知らせタイル

    * 掲示板TOP（NOTICE フィルタ）のショートカット
  * 掲示板タイル

    * 掲示板TOP（ALL フィルタ）のショートカット
  * 運用ルールタイル

    * 掲示板TOP（RULES フィルタ）またはルール専用画面のショートカット

URL やクエリパラメータ（例：`/board?tag=NOTICE`）の詳細は掲示板詳細設計書に委譲し、
本章では「HOME 側がどのフィルタ状態を想定しているか」までを定義範囲とする。

---

## 2.6 本章まとめ

* HOME 画面のコンテンツ領域は、

  * セクションA：最新のお知らせ
  * セクションB：機能タイル 2×3
    の 2セクション構成とする。
* 最新のお知らせは、NOTICE タグ付き掲示のうち、
  テナント設定で指定された件数（1〜3件、デフォルト2件）を HOME 上部に表示する。
* 機能タイルは、

  * 上段：お知らせ／掲示板／施設予約
  * 下段：運用ルール／通知設定／ダミー
    とし、掲示板TOPのフィルタ状態に紐づくショートカットとして設計する。
* 掲示板は、雑談板を持たない「公式情報のハブ」として位置づけ、
  HOME はそのハイライトビュー＋機能ポータルとして機能する。
