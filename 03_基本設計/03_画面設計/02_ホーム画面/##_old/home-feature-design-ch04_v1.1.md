## 4.1 セクションの役割

機能タイルセクションは、住民がアプリの主要機能へ **最短で到達できるナビゲーション領域** として設計する。
シンプルで直感的な 6 タイル構成により、利用者が迷わず目的の機能にアクセスできることを重視する。

本章では、基本設計レベルとして **構造・役割・配置・アイコン方針** のみ定義し、実装詳細（Tailwind・JSX）は後続の詳細設計書で扱う。

---

## 4.2 タイル一覧（MVP 構成）

本セクションで表示するタイルは **6つ** とし、並び順は上段-left → right、下段-left → right の順に固定する。

### **上段（1〜3）**

1. **施設予約**（Facility Reservation）
2. **掲示板**（Board）
3. **回覧板**（Circulation）

### **下段（4〜6）**

4. **お知らせ**（Notices）
5. **通知設定**（Notification Settings）
6. **ダミー（準備中）**（Placeholder）

> ※ タイルはすべて `tenant_id` に基づくユーザ権限・機能有効化設定に従う。
> （機能フラグは詳細設計のスコープ）

---

## 4.3 タイル構成要素（基本設計レベル）

各タイルは以下の要素で構成される：

* **アイコン（Lucide Icons）** ※名称のみ記載
* **機能名ラベル（i18n対応）**
* **タップ領域（ボタン領域）**
* **遷移先 URL（アプリ内パス）** ※詳細設計で定義

### アイコン対応表（名称のみ）

| 機能名  | Lucide アイコン名         | 備考             |
| ---- | -------------------- | -------------- |
| 施設予約 | `Calendar`           | 旧「駐車場」から名称変更済み |
| 掲示板  | `MessageSquare`      | 住民掲示板          |
| 回覧板  | `FileText`           | 文書共有           |
| お知らせ | `Bell`               | 最新情報へのショートカット  |
| 通知設定 | `BellRing` or `Bell` | 詳細設計で確定        |
| ダミー  | `Square`             | Placeholder    |

※ Lucide の strokeWidth や size は **詳細設計書側で規定**。

---

## 4.4 レイアウト方針（基本設計）

機能タイルはグリッドレイアウトとし、デバイスごとに以下の構成とする：

| デバイス       | 配置          | 備考            |
| ---------- | ----------- | ------------- |
| スマートフォン    | **2列 × 3行** | スクロール不要で収まる構成 |
| タブレット / PC | **3列 × 2行** | 最大1024px 中央寄せ |

### 配置原則

* タイルサイズ・余白は HarmoNet Design System の基準に従う（詳細設計対象）
* アイコンはタイル中央上部、ラベルはその下に配置する
* 無効タイル（将来拡張）はグレーアウト表現（基本設計では方針のみ）

---

## 4.5 セクション見出し（機能メニュー）

機能タイル領域の見出しには Lucide の **`Grid2x2`** を使用する。
基本設計では名称のみ定義し、スタイルやサイズは詳細設計で扱う。

見出しラベル（i18nキー）：`home.tiles.heading`

---

## 4.6 i18n（方針のみ）

タイル名は StaticI18nProvider により 3 言語に切替可能とする。基本設計では以下のキーのみ定義する：

* `home.tiles.facility`
* `home.tiles.board`
* `home.tiles.circulation`
* `home.tiles.notices`
* `home.tiles.notification_settings`
* `home.tiles.placeholder`

翻訳 JSON と文言は詳細設計のスコープ。

---

## 4.7 将来拡張方針（Basic）

* タイル追加が必要になった場合は **行数・列数の再設計** を行う
* 7つ以上になる場合はカテゴリ分割（タブ / セクション分割）も検討対象
* 設定画面・アンケート等の追加を想定し、ラベル幅は可変とする

---

## 4.8 本章まとめ

* HOME の主要導線として 6 タイルを配置（MVP仕様）
* Lucide Icons を使用（基本設計では名称のみ）
* 実装詳細（サイズ・Tailwind・JSX）は記載しない
* レイアウトはスマホ 2×3、PC 3×2 の統一仕様
