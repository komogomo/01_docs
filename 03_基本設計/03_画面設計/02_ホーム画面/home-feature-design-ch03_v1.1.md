# HOME 基本設計書 ch03：お知らせセクション

## 3.1 役割（Role）

お知らせセクションは、住民に対して **最新の重要情報を迅速に伝える** ことを目的とした HOME 画面の最上位コンテンツである。

* 重要な情報を見逃さないための “即時可視化”
* 最新 3 件に絞った **シンプルな速報性** の提供
* カテゴリ（重要 / イベント / 連絡 / メンテナンス）による視認性向上
* ユーザーが次に取る行動（詳細確認 / 一覧参照）への導線

本章では、UI の概要と表示ルールのみを基本設計レベルで定義する。

---

## 3.2 表示内容（Display Content）

次の情報を 1 件の「お知らせカード」として表示する：

* **タイトル**（1行、省略表示あり）
* **カテゴリバッジ**（4種：重要 / イベント / 連絡 / メンテナンス）
* **未読バッジ**（未読の場合のみ表示）
* **投稿日**（日付フォーマットは詳細設計で定義）

**表示件数**：常に最新 3 件
データスコープ：ログイン中ユーザーの `tenant_id` のみ

---

## 3.3 インタラクション（Interaction）

* **カードタップ → お知らせ詳細画面（/notices/[id]）へ遷移**
* **「一覧を見る」ボタン → お知らせ一覧画面（/notices）へ遷移**

インタラクション仕様は HOME 詳細設計書で定義し、
基本設計書では「何が起きるか」のみを記載する。

---

## 3.4 UI 構造（基本設計レベル）

HOME の第 1 セクションとして、以下の構造とする：

```
<section>
  ├─ 見出し（"最新のお知らせ"） ※i18n対応
  ├─ お知らせカード × 3
  │    ├─ カテゴリバッジ
  │    ├─ 未読バッジ（未読時のみ）
  │    ├─ タイトル（1行、省略）
  │    └─ 投稿日時
  └─ 「一覧を見る」リンク
</section>
```

* 見出しのアイコンは **Lucide Icons `Bell`** を使用（基本設計では名称のみ）
* レイアウト・スタイルは HOME 詳細設計書で定義（Tailwind 記述はここに含めない）

---

## 3.5 文言・i18n（方針のみ）

基本設計で定義するレベルは以下のみ：

* 見出し：`home.notices.heading`
* 「一覧を見る」：`home.notices.view_all`
* カテゴリ名キー：

  * 重要：`notice.category.important`
  * イベント：`notice.category.event`
  * 連絡：`notice.category.info`
  * メンテナンス：`notice.category.maintenance`

各文言の実体（辞書 JSON / 翻訳）は詳細設計で作成する。

---

## 3.6 アクセシビリティ（概要）

最低限の基本設計要件として：

* 見出しに `aria-labelledby` を使用可能な構造とする
* カードは `<article>` を推奨
* 未読バッジには `aria-label="未読"` を付与する方針
* タイトル省略（ellipsis）時は **スクリーンリーダー用に完全文言を読ませる実装** を詳細設計で行う

---

## 3.7 本章まとめ

* HOME 画面の最上位セクションとして「最新3件のお知らせ」を提示する
* 見出し → カード → 一覧導線 のシンプルな構造
* Lucide の Bell を使用（名称のみ記述）
* i18n・スタイル・タップ挙動の詳細は後続の HOME 詳細設計書で扱う
